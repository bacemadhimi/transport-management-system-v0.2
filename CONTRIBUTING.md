# Règles de contribution – TMS

## 1. Gestion des issues
- La création des issues, tâches, sprints et milestones est réservée au Tech Lead
- Toute issue créée sans validation sera automatiquement fermée

## 2. Développement
- Aucun développement sans issue assignée
- Une branche = une issue
- Convention de nommage :
  feature/ISSUE-ID-description

## 3. Pull Requests
- Toute Pull Request doit référencer une issue
- Le merge sur main est interdit sans validation du Tech Lead

## 4. Discipline d’équipe
- Le non-respect de ces règles entraîne le refus de la PR
